소개

풀스택 개발자 박준용 입니다.

개발 경험 2년 주니어 개발자로, Node.js와 Express를 중심으로 한 웹 서비스 개발 경험을 보유하고 있습니다. React와 Next.js를 활용한 모던 프론트엔드 구현뿐만 아니라, Electron과 PySide를 활용한 데스크탑 GUI 애플리케이션 개발 경험도 있습니다. 또한 WordPress 기반 웹사이트 제작과 MySQL, MongoDB, MariaDB를 활용한 데이터 관리 역량을 갖추고 있어 기획부터 배포까지 프로젝트 전 과정을 수행할 수 있습니다. 문제 해결과 빠른 학습 능력을 바탕으로 팀 협업과 성과 창출에 강점을 가지며, 새로운 기술 도입과 최적화에도 적극적으로 참여합니다.

수강 진도 자동 추적 구현

문제 상황 : 사용자가 강의를 직접 완료해야만 수강 완료로 처리되어, 실제 학습 여부와 진도 추적이 불가능 했습니다.

해결 방법 : 사용자가 강의 페이지에 접속하면 PHP 백엔드에서 수강 상태를 '진행 중'으로 자동 기록하고, JavaScript와 AJAX를 사용해 클라이언트 측에서 학습 시간을 실시간 수집하여 MySQL/MariaDB 진도 테이블에 업데이트했습니다. 이를 통해 사용자가 강의를 빠르게 넘기거나 페이지를 떠나더라도 최소 학습 시간 기준으로 정확한 진도율을 계산하도록 구현했습니다. 또한, 프론트엔드에서는 진도 바를 실시간 반영하여 사용자에게 현재 진행 상황을 직관적으로 보여주도록 개선했습니다.

구현 방식

  • WordPress + learnpress LMS 플러그인 확장
  • PHP로 강의 입장 시점과 종료 시점 기록

  • 사용자가 강의를 건너뛰거나 빠르게 진행할 경우, 최소 학습 시간 기준으로 진도율 계산

결제 시스템 캐싱 문제 해결

문제 상황 : 캐싱 처리로 인해 장바구니와 결제 페이지에서 세션이 불일치하거나, 사용자가 결제를 시도할 때 이전 장바구니 정보가 남아 결제가 실패하는 사례가 발생했습니다.

해결 방법 : PHP 백엔드에서 WooCommerce 장바구니와 결제 세션을 캐싱 예외 처리 대상으로 지정하고, 페이지 로드 시 세션과 캐시를 동기화하도록 구현했습니다. 또한 JavaScript를 사용하여 클라이언트 측에서도 장바구니 상태를 실시간 검증하고, 결제 버튼 클릭 시 서버로 즉시 최신 데이터를 전송하도록 개선했습니다. 이를 통해 캐시로 인한 결제 오류를 방지하고, 사용자 경험을 안정적으로 유지할 수 있었습니다.

대량 요청 처리 및 서버 안정성 확보

문제 상황 : 짧은 URL 생성 요청이 단시간에 몰리거나 봇(자동화된 트래픽)이 대량으로 접근할 경우, 서버 부하가 급격히 증가하고 응답 지연이 발생했습니다.

해결 방법 : 서버 레벨에서 **Rate Limiting(요청 제한)**을 적용하여 단일 IP의 과도한 요청을 차단했습니다. 또한, Nginx 캐싱과 비동기 처리(Queue 기반)를 도입해 순간적인 대량 요청에도 안정적으로 서비스할 수 있도록 개선했습니다.

악성 URL 및 보안 문제

문제 상황 : 사용자들이 단축 서비스를 악용해 악성 사이트나 피싱 URL을 등록하는 문제가 발생했습니다.

해결 방법 : URL 등록 시 화이트리스트/블랙리스트 필터링을 적용하고, 외부 보안 API 연동으로 악성 도메인을 탐지하여 차단했습니다. 또한, IP 로그를 추적하여 동일 IP의 비정상적인 대량 등록을 제한했습니다.

클릭 통계 및 사용자 분석

문제 상황 : 단순 리다이렉션만 제공할 경우, 마케팅이나 사용자 분석에 활용하기 어려웠습니다.

해결 방법 : URL 접속 시 IP, 브라우저, 타임스탬프를 기록하고 이를 기반으로 사용자별 클릭 수와 트래픽 패턴을 시각화할 수 있도록 구현했습니다.

해결 방법 : AJAX 비동기 요청을 적용해 UI가 멈추지 않도록 했고, 서버 측에서는 최근 검색 결과를 임시 저장하여 반복 요청 시 즉시 응답할 수 있도록 구현했습니다.

봇 및 악성 요청 차단

문제 상황 : 일부 사용자가 자동화 스크립트(봇)를 사용해 캐릭터 조회를 반복하여 서버에 과부하를 일으켰습니다.

해결 방법 : IP별 요청 제한을 적용하여 단기간 다수 요청을 차단하고, 비정상적인 반복 접근을 기록하여 서버 안정성을 확보했습니다.

대량 데이터 관리

문제 상황 : 150,000건 이상의 검색 기록이 누적되면서 데이터 조회 속도가 점점 느려졌습니다.

해결 방법 : MySQL 인덱스를 최적화하고 페이지네이션을 적용하여 대량 데이터도 빠르게 조회할 수 있도록 개선했습니다.

보안 및 안정성 강화

해결 방법 : 서버에서 Proxy API를 구현하여 API 키를 안전하게 보호하고, 과도한 요청을 막기 위해 요청 제한 로직을 추가했습니다.

구매자 전용 접근 제어

문제 상황 : 워드프레스 기본 솔트 처리 외에 LMS 플러그인에서 추가된 런프레스+워드프레스 솔트 때문에, 기존 워드프레스 로그인 검증만으로는 구매자 인증이 불가능했습니다.

해결 방법 : 서버 측에 전용 API를 구현하여 워드프레스 로그인 정보를 직접 검증하도록 변경했습니다. 또한, 클라이언트에 노출되면 안 되는 키 값들은 .env 환경 변수로 안전하게 관리하고, 서버에서만 불러와 사용하도록 처리하여 보안성을 강화했습니다.

코드 보호 및 난독화

문제 상황 : 구매자가 Electron 앱을 언패킹하여 내부 JS 코드를 열람할 가능성이 있었습니다. 이 경우 민감한 로직이나 API 키가 노출될 위험이 있었습니다.

해결 방법 : 클라이언트 코드에 대해 난독화 및 ASAR 패키징을 적용하여 역공학을 방지했고, Secrets Management 전략을 도입하여 API 키는 .env 환경 변수로 분리 및 서버 측에서만 관리되도록 개선했습니다.

자동 업데이트 기능

문제 상황 : 배포 후에도 기능 개선이나 버그 수정이 필요했지만, 사용자가 직접 업데이트해야 하는 번거로움이 있었습니다.

해결 방법 : GitHub Release API와 Electron autoUpdater 모듈을 연동하여, 프로그램 실행 시 최신 버전이 자동으로 다운로드 및 적용되도록 구현했습니다.

해결 방법 : Electron의 크로스 플랫폼 빌드 기능을 활용하여 OS별 빌드를 생성하고, 공통 UI/UX를 유지할 수 있도록 최적화했습니다.

CAPTCHA 및 사용자 입력 대응

문제 상황 : 반복적인 포스팅 과정에서 CAPTCHA가 자주 나타나 자동화 작업을 방해하고, 다양한 입력 형식으로 인해 프로그램이 멈추거나 오류가 발생했습니다.

해결 방법 : 2Captcha 서비스를 연동하여 CAPTCHA를 자동으로 해결하도록 구현했습니다. 또한, 실제 사용자가 작성한 것처럼 보이도록 Gemini API에서 수집한 데이터를 활용해 입력값을 동적으로 생성, 프로그램이 자연스럽게 포스팅되도록 개선했습니다.

반복 포스팅 중 안전성 보장

문제 상황 : 자동화 반복 포스팅 시, 동일 데이터 중복 전송이나 로직 오류로 인해 게시물이 잘못 올라가는 문제가 발생할 수 있었습니다.

해결 방법 : 프로그램 내 입력 검증 및 포스팅 로그 관리 시스템을 구축하여, 이전 데이터와 중복되는 항목을 자동으로 체크하고, 오류 발생 시 롤백할 수 있는 안전 장치를 구현했습니다.

효율적인 자동 클릭 및 세션 관리

문제 상황 : Selenium으로 반복적으로 클릭하며 페이지에 접근하면, 처리 속도가 느려지고 불필요한 작업이 많았습니다. 또한, 로그인 세션이 만료되면 자동화가 중단될 위험이 있었습니다.

해결 방법 : 필요한 URL과 입력 값을 미리 읽어와, 한 번에 해당 페이지로 바로 접근하도록 구현했습니다. 쿠키를 저장하여 로그인 상태를 유지하고, 세션 만료 시에는 자동으로 재로그인하도록 로직을 추가하여 안정성과 효율성을 높였습니다.

서버 기반 HTTP 요청 자동화

문제 상황 : GUI를 계속 켜두고 Selenium으로 작업하면 리소스 소모가 크고 속도가 느렸습니다. 대량 포스팅 시 효율성이 떨어지는 문제가 있었습니다.

해결 방법 : AWS EC2에서 Python과 JS를 활용하여 페이지를 직접 분석하고, Selenium 없이 HTTP 요청으로 자동 포스팅이 가능하도록 구현했습니다. 이를 통해 GUI를 켜두지 않아도 빠르고 안정적으로 포스팅을 수행할 수 있게 최적화했습니다.

IP 회전 및 우회 크롤링

문제 상황 : 자동 포스팅과 데이터 수집 과정에서 동일 IP로 반복 요청 시 사이트 차단이 발생할 수 있었습니다.

해결 방법 : Smartproxy 회전 프록시를 연동하여, 요청 시마다 IP를 자동으로 변경하도록 구현했습니다. 이를 통해 크롤링과 포스팅의 안정성을 확보하고, 대량 데이터를 지속적으로 처리할 수 있었습니다.
